1. git checkout -- <file>
2. git restore --staged <file>
3. git reset HEAD~
4. git checkout -b <branchname>
5. Fast-forward merge происходит, когда история коммитов целевой ветки является прямым продолжением истории нынешней ветки. При таком слиянии не создается новый коммит слияния, а указатель нынешней ветки попросту перемещается на окончательный коммит целевой ветки. Не fast-forward merge применяется, когда истории веток разделяются на некотором уровне, и создается новый коммит слияния, который объединяет видоизменения обеих веток.
6. Конфликт слияния возникает, когда две ветки имеют разные изменения в одной и той же области кода. Git не может автоматически определить, какие изменения следует применить, поэтому он помечает файлы с конфликтами. Разрешение конфликта включает в себя ручное редактирование файлов с конфликтами, чтобы выбрать нужные изменения или объединить их вручную.
7. git branch -D <branchname> , git branch -d <branchname>
